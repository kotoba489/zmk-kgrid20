ファームウェアは左手だけ
ご安心ください、右手のキーマッピングも自由自在に変えられます！
少し不思議に感じるかもしれませんが、ZMK（および多くの分割キーボード）には
**「脳みそ（親機）は片方にしかない」**というルールがあります。

なぜ左手だけ書き換えればいいのか？
分割キーボードにおける「キーマップ（どのキーが何の文字か）」のデータは、すべて**親機（左手）**が管理しています。
右手の役割: 「この場所のボタンが押された」という物理的な信号を左手に無線で飛ばすだけです。
左手の役割: 右手から届いた「この場所のボタン」という情報を受け取り、設定ファイル（.keymap）を見て「それは 'L' の文字だな」と判断して、Macに送信します。
つまり、右手の動きを司っているのは、左手の中にある設定ファイルなのです。

今後のキーマップ変更の流れ
GitHub上の .keymap ファイルを編集する（左手分も右手分も一緒に書き換えます）。
Commit & Push してビルドする。
left.uf2 だけをダウンロードして、左手に書き込む。
完了！ 右手には一切触れていないのに、右手のキー配置が新しくなっています。


結論：右手を書き換えるときはいつ？
右手にファームウェアを書き直す必要があるのは、今後**「根本的な通信の仕組み」や「省電力設定」**などを変えた時だけです。
「AのキーをBに変えたい」「レイヤーを増やしたい」といったキー配置の変更であれば、今日からはずっと左手の書き換えだけで済みます。
